162. Find Peak Element

问题描述

https://leetcode.com/problems/find-peak-element/

A peak element is an element that is greater than its neighbors.

Given an input array nums, where nums[i] ≠ nums[i+1], find a peak element and return its index.

The array may contain multiple peaks, in that case return the index to any one of the peaks is fine.

You may imagine that nums[-1] = nums[n] = -∞.

峰值元素是指其值大于左右相邻值的元素。

给定一个输入数组 nums，其中 nums[i] ≠ nums[i+1]，找到峰值元素并返回其索引。

数组可能包含多个峰值，在这种情况下，返回任何一个峰值所在位置即可。

你可以假设 nums[-1] = nums[n] = -∞。

思路

  • 这题一开始没想出做法，看了一下网上的解答，发现是没有充分利用起nums[-1] = nums[n] = -∞这个隐含的条件。
  • 因为左右两边都是-∞，所以如果阵列是单调递增的，那么极大点就在右端点，如果阵列式单调递减的，那么极大点就在左端点。如果两个都不是，极大点肯定就在中间。故峰值一定在区间中。
  • 可以用二分法进行搜索，找到一个中点nums[mid]，如果nums[mid+1]比它大，那么nums[mid+1]或它的右边必然有峰值。反之，nums[mid]或它的左边必然有峰值。

Find the Duplicate Number

问题描述

https://leetcode.com/problems/find-the-duplicate-number/

Given an array nums containing n + 1 integers where each integer is between 1 and n (inclusive), prove that at least one duplicate number must exist. Assume that there is only one duplicate number, find the duplicate one.

给定一个包含 n + 1 个整数的数组 nums，其数字都在 1 到 n 之间（包括 1 和 n），可知至少存在一个重复的整数。假设只有一个重复的整数，找出这个重复的数。

思路

  • 这题也是一开始没想出做法，看了一下网上的解答，发现是没有充分利用起each integer is between 1 and n (inclusive)和there is only one duplicate number这个隐含的条件。
  • 也就是说数组里有n+1个数，数的范围是1~n。因为只有1个数重复，所以每一个值都会取到，其中有一个值被取到了两次
  • 抽屉原理。n+1个下标相当于苹果，n个值相当于抽屉。把值域（抽屉）分成左右两个半边，其中必有一个半边多了一个苹果。只要找到多了一个苹果的半边，不断二分就能定位出最后的抽屉。

0 1 背包

内容

  • N 个物品，总容量不能超过 V
  • 每个物品体积是vi，价值是wi
  • 每件物品至多只能用一次

思路

从两个方面来思考：

  • 状态表示 f(i,j)
    • 集合
      • 满足下列条件的所有选法
      • 条件
        • 只从前i个物品中选
        • 总体积不超过j
    • 属性
      • Max
      • Min
      • Cnt
  • 状态计算
    • f(i,j) 集合划分
      • 原则：不重、不漏
      • 不含 i 的放一边
        • f(i-1,j): 从1～i-1选，且总体积不超过j的选法
      • 含 i 的放一边
        • f(i,j): 从1～i选，且总体积不超过j的选法
        • 相当于 f(i-1,j-vi) + wi

优化成一维数组

原理：计算dp[i][j] = max(dp[i-1][j-vi]+wi, dp[i-1][j]) 的时候其实只用到了 i-1 一层，和第 i 层没关系

完全背包问题

内容

  • N 个物品，总容量不能超过 V
  • 每个物品体积是vi，价值是wi
  • 每件物品可以用无限次

思路

从两个方面来思考：

  • 状态表示 f(i,j)
    • 集合
      • 满足下列条件的所有选法
      • 条件
        • 只从前i个物品中选
        • 总体积不超过j
    • 属性
      • Max
      • Min
      • Cnt
  • 状态计算
    • f(i,j) 集合划分
      • 按照第i个物品选了几个来划分
      • 0 1 2 3 … k-1 k
    • 曲线救国
      • 去掉k个物品i
      • 求 Max(f[i-1,j-k*v[i]])
      • 再加回来 k 个物品 => f[i-1,j-k*v[i]] + w[i]

优化版

思路：

  • f[i,j] = Max(f[i-1,j], f[i-1,j-v]+w, f[i-1,j-2v]+2w, f[i-1, j-3v]+3w, ...)
  • f[i,j-v] = Max(f[i-1,j-v], f[i-1,j-2v]+w, f[i-1,j-3v]+2w, f[i-1, j-4v]+3w, ...)
  • f[i,j] = Max(f[i-1,j], f[i,j-v]+w)
  • 只需要枚举两个状态

强类型与弱类型

穷人翻身真的是因为高考吗

Diposting di 2020-07-06

又到了一年一度的高考的日子。说起高考这个话题，有一个很常见的论点是

高考是窮人唯一翻身的機會

每个持“高考翻身”论的人具体表述可能不同，但意思差不多就是如此。但关于“穷人通过高考翻身”这个论点，我认为还是有必要探讨几句。

首先是这句话的具体所指。穷人，关于贫困的定义虽然具体的数值标准不一，但对穷人的概念基本没什么歧义，直观理解就行。至于翻身这个词，我倾向于定义成阶层的上升。或者更加直观地看，就是收入显著增加。

进入正题。回顾历史，不难发现这句话普遍成立的时代主要是八、九十年代。在那个时候，大量的农村穷人通过高考进入大专或者大学，然后在城市里找到体面的工作，实现成为中产阶层的阶层上升。也就是说，八九十年代穷人翻身这个现象是客观存在的。但这一现象与高考的联系——是不是穷人翻身真的只是因为高考呢，这就必须仔细探讨了。

我在这里先阐明观点。

八、九十年代穷人通过上述路径实现阶级上升，根本原因不是高考，而是改革开放带来的政策红利。

在进一步论述之前，先阐述一下“穷人翻身”这一现象的机制。改革开放带来了中国经济的高速发展，随之而来的是更多的就业岗位和相对更高的薪资水平，尤其是城市。新出现的岗位中，有不少都需要一定的知识、技能。这样的知识、技能往往需要在大专院校里习得，因此考上大专、大学，才有机会在城市里得到这样的体面工作。而高考仅仅是作为选拔机制，选拔出农村里的人才读大专大学，从而使他们得到这样的工作。这就是所谓“穷人翻身“的机制。

在理解了这样的机制以后，我们不难发现：

经济改革在城镇提供的大量新增岗位才是“穷人翻身”的基础。

经济改革在城镇提供的大量新增岗位，这些新增岗位本身就必须要有人做，区别只是谁来做而已。其实仅仅是从个体的角度来说有区别，但从社会的角度看区别并不大。通过高考来决定谁从事这样的岗位是一个不错的选拔机制，但夸张一点说，就算是摇号抽签分配这些岗位，也同样能起到让不少穷人“翻身”的结果。

说到经济改革和城镇的新增岗位，需要理解一个时代背景：在计划经济体制下，城镇居民和城镇岗位是一个萝卜一个坑的关系，城市劳动力市场并是今天这样的买方市场（或者说根本不存在市场）。所以在大规模国企改革前，城镇原有的居民一方面基本上不属于“穷人”的范畴，另一方面基本上不会去抢这些新增岗位的饭碗。所以会争夺这些新增岗位的人，本身就是来自农村的“穷人”。而高考归根结底在这里发挥的作用——再一次说明——不过就是争夺新增资源（岗位）的手段而已。

所以说到这里，该阐述的部分已经都阐述完了。但结合前述时代背景，我们还可以得出一个推论：

一旦新增的岗位逐渐减少，而城镇里成长起来的市场经济一代（which means 这些人不能再继承父辈的铁饭碗）带着优渥的教育资源加入这场分配新增岗位的厮杀的时候，还能通过高考实现翻身的穷人将是凤毛麟角

而这，就是当下的现状。
